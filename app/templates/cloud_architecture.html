Cloud architecture

Every security control in this platform maps directly to a managed cloud-native primitive. The page below is intended for reviewers, auditors, and engineers; ordinary patients and clinicians never need to see it.

Security ↔ cloud mapping

Security requirement Cloud service Why the cloud version is stronger
EHR confidentiality at rest S3 with SSE-KMS, bucket versioning Server-side encryption with managed key rotation; versioning protects against tamper / ransomware rollback
Per-patient key isolation AWS KMS (envelope encryption) One CMK wraps every per-patient data key; revoking the CMK revokes all decryption ability
Authentication Amazon Cognito user pool Hosted identity service; pluggable MFA; tokens issued and signed by AWS
Authorization (RBAC + scoped grants) IAM-style policy evaluated in AWS Lambda Declarative JSON policy; least-privilege ladder (read < write < history)
Provider verification External CMS NPI Registry API Authoritative source of truth — admins authorize the lookup, they don't decide who is a real doctor
Tamper-evident audit DynamoDB chained-hash ledger + CloudTrail Two independent records of the same events; chain breaks are visible in seconds
JWT signing key custody AWS Secrets Manager Never in code; supports automatic rotation; access is itself audited
Patient export integrity AWS KMS asymmetric (RSA-2048) signing Audit PDF carries a cryptographic signature anyone can verify with the public key
Anomaly detection CloudWatch Logs metric filter + alarm Failed-access bursts trigger an automated alarm
Reduced compute exposure API Gateway in front of Lambda No EC2/container surface; throttling and WAF available
Disaster recovery S3 versioning Point-in-time restore on any object
Account protection bcrypt password hashing + lockout policy Five failed sign-ins lock the account for 15 minutes; passwords are never stored in plaintext

Why every box is a cloud service

Replacing any service in this design with a "local" alternative (a flat file, an environment variable, an in-process dictionary) weakens at least one row of the mapping above. Keeping every control in a managed cloud primitive is what lets a small team operate this platform under HIPAA without standing up its own key-management, identity, audit, and monitoring infrastructure.

Demo deployment

For development and academic demos this stack runs on LocalStack, which provides API-compatible local emulators for the AWS services above. The application code is identical between LocalStack and a real AWS deployment — only the endpoint URL changes.
